## カテゴリとタグ

### カテゴリ

記事ごとに以下の7カテゴリから必ず1つだけ設定する。

- ブログ運営と副業
- もみじと暮らす
- 備えと暮らし
- 癒しと散策
- 社会・時事・思考
- 趣味・体験・その他
- 食と懐かしさ

### タグ

- 記事ごとに3〜5個を必ず設定する。
- 記事内容から適切なキーワードを抽出する。
- 単発の固有名詞だけでなく、カテゴリ横断で使える汎用タグも含める。
- 既存タグとの統一感を優先し、新規タグを乱造しない。
- 例：柴犬、ラーメン、八王子、50代、副業、相模原。

### 「ブログをアップして」時の動作

1. 対象記事のカテゴリ・タグを確認する。
2. 不足があれば記事内容から自動補完する。
3. 補完内容を報告して、ひろしさんの確認を得る。
4. OK確認後に `git add` → `git commit` → `git push` する。

記事編集時もカテゴリ・タグを確認し、不足があれば補完して報告する。

## 記事スタイル（blog-writer-ossan 相当）

### 語り手

- 関西出身・50代男性。
- 一人称は「僕」。
- 標準語ベースで、語尾だけほんのり関西弁。
- OK：`〜やね`、`〜かなぁ`、`〜やと思う`、`〜やけど`。
- NG：コテコテの漫才口調、`なんでやねん`、ネットスラング。

### 鉄則

- 入力された事実のみで書く。
- 架空エピソードや脚色は絶対にしない。
- 五感描写は入力素材にある事実だけを使う。
- 「〜すべき」「正解はこれ」と断定しない。
- データや有益情報は記事前半に置く。
- 他者貢献を最重要視する。

### 基本構成

ブログ記事の本文構成は、日本語記事・英語記事ともに以下の順番を基本形として恒久適用する。

1. 読了時間
2. 冒頭（共感フレーズ＋体験の予告）
3. こんな人に読んでほしい
4. 結論（気づき。断定しない）
5. 目次
6. 本文（時系列または心の動き順。自虐やかっこ悪さも隠さない）
7. まとめ（静かに締める。答えを出さず、余韻で終わる）
8. 内部リンク3本

タイトルは frontmatter の `title` に設定し、30文字前後で煽らない。

### 入力素材

記事を書く時は、可能なら以下を受け取る。

- 出来事（何が起きたか、どこで、いつ頃）
- その時に感じたこと（不安、迷い、違和感など）
- 五感メモ（事実だけ、短く）
- あとから気づいたこと・今の正直な気持ち
- 読者にそっと伝えたいこと（答えではなくヒント）

## 画像ルール

### 必須画像

記事ごとに以下を想定する。

- アイキャッチ画像
- SNS画像
- 各章画像（章ごとに1枚）
- まとめ画像

### frontmatter必須項目

- `eyecatch` と `sns_image_url` は全記事で必ず設定する。
- `sns_image_url` はX投稿・OGP表示で使うSNS用画像として扱う。
- 専用SNS画像がない場合でも、記事内容に合う `eyecatch` または本文内の代表画像を `sns_image_url` に設定し、未設定のままにしない。
- SNS画像URLが壊れている、HTMLを返す、画像ファイルが存在しない状態で公開・投稿しない。

### 基本方針

- ひろしさんが撮影した実写を優先する。
- 実写の有無を毎回確認せず、添付・指定ディレクトリ・会話内パスを自動確認する。
- ハイキング、散歩、もみじ、ラーメン、旅行、グルメは実写中心。
- 副業、家計、通信費、思考、社会、病院、健康はAI生成中心。
- 写真が合わない章や写真がない章はAI生成を検討する。

### ファイル名

- `[記事スラッグ]-eyecatch.jpg` または `.webp`
- `[記事スラッグ]-sns.jpg`
- `[記事スラッグ]-[内容を表す英単語].jpg` または `.webp`
- `[記事スラッグ]-summary.jpg`
- 言語別画像が必要な場合は `-ja` / `-en` を付ける。

### 保存先

- 基本：`/images/wp-content/uploads/[年]/[月]/`
- 実体配置は `public/images/` 配下の既存構成に合わせる。

### 日本語版・英語版

- AI生成画像に日本語テキストが含まれる場合、英語版用に英語テキスト版を別途作る。
- テキストなしの画像は日英共通でよい。
- 実写は基本的に日英共通で使用する。

## X投稿ルール

投稿前チェックリストは全件必須。

- 投稿スケジュールは、日本語ブログを 07:30 / 12:30 / 21:00（JST）、英語ブログを 22:00 / 翌02:00 / 翌11:00（JST）にする。
- SNS画像（`-sns.jpg`）を添付する。
- 短縮URL（`tinyurl.com`）を含める。
- ハッシュタグは3〜5個。
- 日本語投稿は140字以内、英語投稿は280字以内。
- 1つでも欠けたら投稿しない。

文字数調整：

- URLは11.5文字固定カウント。
- 5〜10字オーバー：冗長表現を削る。
- 10〜25字オーバー：修飾語・接続詞を削る。
- 25字以上オーバー：要約再構成、タグ削減。ただし最低3個維持。

ログ：

- ログファイル：`~/blog-x-post-log.md`
- 記録項目：投稿日時、記事タイトル、X投稿URL、短縮URL、ハッシュタグ、文字数、調整段階、SNS画像ファイル名、結果。

## デプロイ

- 通常は `npm run build` で確認してからコミット・pushする。
- `git push` だけでは本番に反映されない。Pages の本番反映は必ず `npm run build && npx wrangler pages deploy dist --project-name freelife50-astro` で行う。
- 本番影響のある変更、ビルドエラー、デプロイ失敗は具体的なエラー内容を報告して止める。

## Cloudflare構成の注意事項

- `freelife50-cache-bypass` はスタンドアロンWorkerとして `freelife50.com/*` と `en.freelife50.com/*` の両方を intercept している。
- このWorkerは全リクエストを `freelife50-astro.pages.dev` に転送し、`X-Original-Host` ヘッダーで元ホスト名を Pages 側に渡す。
- このWorkerでは `redirect: "manual"` を維持し、Pages 側のリダイレクトをクライアントへそのまま返す。
- `freelife50-astro` は Cloudflare Pages Advanced Mode の `dist/_worker.js` で動作し、`X-Original-Host` によって `freelife50.com` と `en.freelife50.com` を判別する。
- `en.freelife50.com/` は `/en/index.html` にリライトし、`freelife50.com/[en-slug]` は `en.freelife50.com` にリダイレクトする。
- Worker Routes（`freelife50.com/*` / `en.freelife50.com/*`）を削除しない。
- `freelife50-cache-bypass` から `X-Original-Host` ヘッダーを削除しない。
- `freelife50-cache-bypass` から `redirect: "manual"` を削除しない。
- PagesではなくWorkersとしての `wrangler deploy` は実行しない。
- Cloudflare側に何か変更を加えた場合は、その内容を必ずコミットメッセージに記録する。
